Fitur

Semua yang kamu butuhkan.
Tidak lebih, tidak kurang.

Setiap fitur dipilih berdasarkan apa yang benar-benar dipakai saat testing.

Request Builder

Visual, bukan kode

Isi method, URL, headers, dan body lewat form intuitif. Tidak perlu hafal format cURL atau syntax JSON.

POST /v1/auth/login
Authorization: Bearer {{token}}
Content-Type: application/json
Variable Extraction

Ambil data antar request

Simpan nilai dari response seperti token atau ID lalu gunakan di request berikutnya — otomatis, tanpa script.

Login → simpan {{token}} dari data.token
Get Cart → pakai {{token}} di header
Checkout → pakai {{cart_id}}
Assertions

Validasi otomatis, hasil jelas

Tentukan ekspektasi untuk status code, isi response, dan waktu. Setelah send langsung tahu pass atau fail.

✓ PASS Status code = 200
✓ PASS Response time < 500ms
✗ FAIL data.user.email exists
Environment

Dev, Staging, Prod — 1 klik

Simpan base URL dan variabel per environment. Switch antara Dev, Staging, dan Production tanpa edit manual.

Dev    → api.dev.myapp.id
Staging → api.stg.myapp.id
Prod   → api.myapp.id

FAQ

Pertanyaan yang sering ditanya.

Apakah tier gratis benar-benar gratis selamanya? +
Ya. Tidak ada batas waktu, tidak perlu kartu kredit. Upgrade hanya jika kapasitas kamu bertumbuh dan ingin mendukung pengembangan HITSaurce.
Apakah perlu install sesuatu? +
Tidak sama sekali. HITSaurce berjalan sepenuhnya di browser — tidak perlu ekstensi, tidak perlu desktop app, tidak perlu Node.js atau Python.
Bagaimana cara kerja load test dengan K6? +
Kamu mengatur jumlah virtual user dan durasi lewat UI. HITSaurce menjalankan K6 di server kami dan streaming hasilnya secara real-time ke browser — tanpa install K6 lokal sama sekali.
Bisa import collection dari Postman? +
Ya, import collection Postman format JSON tersedia di semua tier termasuk gratis. Kamu bisa langsung pakai collection yang sudah ada tanpa setup ulang.
Apakah data request saya aman? +
Semua komunikasi dienkripsi via HTTPS. Data kamu tidak pernah dijual atau dibagikan ke pihak lain. Request yang kamu buat hanya bisa dilihat oleh kamu dan anggota workspace-mu.
Apakah bisa dipakai untuk QA yang belum tahu coding? +
Justru itu target utama kami. HITSaurce dirancang agar QA manual bisa langsung testing API tanpa perlu belajar coding, cURL, atau Postman dulu. Semua visual dan ada panduan inline di setiap langkah.
